Routing - Using a RouteToLabel node to route data in a message flow

IBM Knowledge Center
Learn how to use the RouteToLabel node to route messages in a message flow driven by a JSON message over HTTP.
  • Create
  • Prepare
  • Run

Import projects

When you click Import, one project will be created in your workspace: An application named ExampleRoutingRouteToLabel.

ExampleRoutingRouteToLabel provides a single message flow named RoutingRouteToLabel which uses an HTTPInput node to receive a JSON input message, a Mapping node and RouteToLabel nodes to route the message, several Label nodes each of which feeds a FileOutput node (configured to write to a specific file name) and an HTTPReply node.

Once imported using the link on the right, the provided project is ready to be deployed and tested. To deploy, you can use a provided BAR file, as described on the Prepare tab.

Prepare an output directory, create an integration node and deploy the message flow

Prepare the scenario by executing the following steps:

  1. The message flow's FileOutput nodes will attempt to write output files to the directory C:\temp. Create this directory if it does not already exist, or edit all the FileOutput nodes in the message flow so that they write to a different directory suitable for your runtime environment.
  2. Next, create an integration node by starting an App Connect Enterprise command console and executing the following commands:
    mqsicreatebroker TESTNODE
    mqsistart TESTNODE
    mqsicreateexecutiongroup TESTNODE -e server
  3. Back in the Toolkit, drag and drop deploy the provided BAR file named RoutingRouteToLabel.bar which you will find inside the ExampleRoutingRouteToLabel application.

Use the Flow Exerciser to test the tutorial scenario

  1. Open RoutingRouteToLabel.msgflow.
    • Click the Flow Exerciser icon to start testing the flow (when challenged you do not need to redeploy the flow again).
    • Click the Send Message icon .
    • Four saved messages are provided. Each message provides a JSON structure which carries the FirstName, LastName and Country for a fictitious person. Select InputMessageJoeBloggs and click Send and your message is sent to the HTTPInput node.
    • InputMessageJoeBloggs contains this data:
      {"FirstName":"Joe","LastName":"Bloggs","Country":"GB"}
    • After you close the dialog, the paths taken through the messageflow are highlighted. Depending on the value of the Country field, the Route node should route the data to the correct instance of a FileOutput node whose role is to write the data to an output file with the right country name. For example, InputMessageJoeBloggs should be written into CountryGB.json

This tutorial has shown the combination of a Mapping node and a RouteToLabel node making message routing decisions.
